Kanji Yomigana · Privacy & Data Handling

This page is the extension's built-in privacy notice. For Chrome Web Store publication, it is still a good idea to host a matching privacy policy on a public URL as well.

What this extension does

Kanji Yomigana helps users read Japanese pages by showing context-aware yomigana when a user clicks a kanji or Japanese expression.

Data sent to external AI providers

  • The Japanese expression that the user clicked
  • A short surrounding context needed to infer the correct reading
  • A short display helper surface when okurigana or connected kana are needed for a natural result

The destination provider is chosen by the user: Anthropic, Google Gemini, or OpenAI. The extension uses a BYOK (Bring Your Own Key) model, meaning the user directly enters their own API key.

Data stored locally in the browser

  • Lookup history used for the Today and Focus views
  • Response cache used to reduce repeated requests, latency, and cost
  • User settings such as provider choice, retention periods, site access rules, and UI language

The default mode is minimized storage. URL, page title, and context are not stored unless the user explicitly enables extended metadata storage.

API key storage modes

  • Session only: the key is removed when the browser session ends.
  • Remember on this device: the key remains in local browser storage across restarts.

After saving, the full API key is not shown again in the popup. Only a masked suffix is displayed.

User controls

  • Choose Session only or Remember on this device for API key storage
  • Enable or disable extended metadata storage
  • Configure history and cache retention periods
  • Control where the extension runs with allow/block site lists
  • Use Delete all data to remove keys, settings, history, cache, and dismissed items

Third-party providers

  • Anthropic Claude API
  • Google Gemini API
  • OpenAI API

Please also review the privacy and data handling policies of the provider you choose to use.
